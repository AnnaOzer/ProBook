docker images
покажет, какие образы контейнеров у меня на машине находятся, скачаны
имя образа состоит из двух частей:
- репозиторий, в целом имя образа
- тег, как версия образа
- и есть у образа ID

docker pull
пример
docker pull php:8.1-cli
- до двоеточия имя самого образа
- после двоеточия тег, т.е. версия
команда в этом примере скачает образ с docker hub на локальную машину
этот образ - по сути архив с файлами, которые понадобятся php 8.1-cli для того,
чтобы в этом контейнере у меня php запустился

docker rmi ID_образа
удалить образ из локального репозитория
(когда он у меня на машине, говорю "локальный репозиторий")
опять docker images и видим, что тот образ стёрт (~полгига освободил)

что есть что:

- образ это архив с файлами

- контейнер это процесс, который будет запускаться;
ему эти файлы (образ) будут подсунуты как та файловая система, которую он увидит,
он ничего кроме них не увидит
- в этих файлах (образ) содержится необходимое для корректного запуска процесса,
(например, nginx и его конфиги)
эти файлы систему файловую на моем компьютере не загрязняют
- проблема деинсталляциии не стоит, целиком скачанный образ удаляется, если не нужен

- репозиторий это место, где образы лежат
локальный это моя машина
удаленный это docker hub или свой репозиторий
