docker и docker-compose применять в повседневной жизни как разработчики на php

хотим, чтобы для каждого проекта собралось свое окружение:
- своя версия php
- свои настройки
- своя коллекция расширений

были придуманы контейнеры:
давайте возьмем процесс, который в нашей операционкой работает, php, например, и изолируем его
- запретим общаться с другими процессами
- подсунем фейковую файловую систему
- свою сеть ему
- сделаем так, что каждый процесс думал, что он один единственный выполняется

так мы получим контейнер

docker - система управления такими контейнерами:
- не единственная
- популярная
- работает

docker позволяет нужный Вам процесс (nginx, php, базу данных, redis)
- запустить в изолированном окружении
- это позволяет 2 версии php запустить одновременно
- упаковывать, доставлять, развертывать контейнеры

образ
- представьте себе как zip файл, в котором файлики лежат
- это не просто zip файл, а zip файл с историей
- образ подключается к контейнеру при старте таким образом,
что контейнер видит только эти файлы
- к файлам в вашей операционной системе приложение доступа иметь не будет,
только к файлу с образом

есть много консольных утилит с docker
